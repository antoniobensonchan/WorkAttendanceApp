# 数据库结构更新 - 添加公司表

## 📅 更新日期
2026-04-18

---

## ✨ 更新内容

### 新增功能
添加了独立的**公司管理模块**，将公司信息从员工表中分离出来，实现规范化的数据库设计。

---

## 🗄️ 数据库结构变更

### 新增表：companies（公司表）

```sql
CREATE TABLE companies (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL UNIQUE
)
```

**字段说明：**
- `id`: 公司ID（主键，自增）
- `name`: 公司名称（唯一，不可重复）

---

### 修改表：workers（员工表）

**之前的结构：**
```sql
CREATE TABLE workers (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL UNIQUE,
    company TEXT NOT NULL  -- ❌ 直接存储公司名字符串
)
```

**新的结构：**
```sql
CREATE TABLE workers (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL UNIQUE,
    company_id INTEGER NOT NULL,  -- ✅ 改为外键关联
    FOREIGN KEY (company_id) REFERENCES companies(id)
)
```

**改进优势：**
1. ✅ **数据规范化** - 避免公司名重复存储
2. ✅ **数据一致性** - 通过外键约束保证数据完整性
3. ✅ **易于维护** - 修改公司信息只需更新一处
4. ✅ **查询优化** - 可以通过JOIN高效查询

## 📊 受影响的查询

### 1. 考勤记录查询
**更新前：**
```sql
SELECT a.id, w.name as worker_name, w.company, s.name as site_name
FROM attendance a
JOIN workers w ON a.worker_id = w.id
JOIN sites s ON a.site_id = s.id
```

**更新后：**
```sql
SELECT a.id, w.name as worker_name, c.name as company_name, s.name as site_name
FROM attendance a
JOIN workers w ON a.worker_id = w.id
JOIN companies c ON w.company_id = c.id  -- ✅ 新增公司表JOIN
JOIN sites s ON a.site_id = s.id
```

### 2. 员工统计查询
**更新前：**
```sql
SELECT w.id, w.name, w.company, ...
FROM workers w
LEFT JOIN attendance a ON w.id = a.worker_id
GROUP BY w.id, w.name, w.company
```

**更新后：**
```sql
SELECT w.id, w.name, c.name as company_name, ...
FROM workers w
JOIN companies c ON w.company_id = c.id  -- ✅ 新增公司表JOIN
LEFT JOIN attendance a ON w.id = a.worker_id
GROUP BY w.id, w.name, c.name
```

## 🔄 数据迁移

### 自动处理
系统已自动备份旧数据库文件：
- **备份文件**: `attendance_backup.db`
- **新数据库**: `attendance.db`（全新创建）

### 手动迁移（如需要）
如果需要保留旧数据，可以执行以下SQL：

```sql
-- 1. 从旧数据提取唯一公司名
INSERT INTO companies (name) 
SELECT DISTINCT company FROM old_workers;

-- 2. 迁移员工数据，关联公司ID
INSERT INTO workers (name, company_id)
SELECT ow.name, c.id
FROM old_workers ow
JOIN companies c ON ow.company = c.name;
```

## 🎯 优势总结

### 数据规范化
| 方面 | 之前 | 现在 |
|------|------|------|
| 数据存储 | 公司名重复存储 | 公司名单独存储 |
| 数据冗余 | 高 | 低 ✅ |
| 数据一致性 | 可能出现不一致 | 外键约束保证 ✅ |
| 修改成本 | 需更新所有相关员工 | 只需更新公司表 ✅ |

### 查询性能
| 操作 | 之前 | 现在 |
|------|------|------|
| 添加员工 | 简单 | 需先查公司ID |
| 查询员工 | 直接读取 | 需JOIN ⚠️ |
| 修改公司名 | 需批量更新 | 单条更新 ✅ |
| 统计分析 | 字符串分组 | ID分组更高效 ✅ |

---

## ⚠️ 注意事项

1. **必须先添加公司** - 添加员工前需要先有公司数据
2. **外键约束** - 不能删除有员工关联的公司
3. **唯一性** - 公司名不可重复
4. **数据备份** - 旧数据已备份到 `attendance_backup.db`
