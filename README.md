# 企业工地考勤管理系统

## 📋 项目简介

一款**局域网多人共享版**企业工地考勤管理系统，纯本地离线运行，无外网依赖，支持多台电脑同一局域网共享统一数据。

### ✨ 核心特性

- 🍎 **Material Design UI风格** - 现代化、多彩、专业的界面设计
- 🌐 **局域网共享** - 一台电脑作为服务器，其他电脑通过浏览器访问
- 💾 **纯本地运行** - 无需外网，数据完全本地化存储
- 👥 **多人协作** - 多用户可同时操作，数据实时同步
- 🌍 **多语言支持** - 支持英语、简体中文、繁体中文三种语言切换
- 📊 **数据统计** - 员工统计看板、工地统计看板、个人员工统计
- 🔒 **数据安全** - SQLite 单文件数据库，本地存储

## 🚀 快速开始

### 1. 环境要求

- Python 3.10+
- 同一局域网内的多台电脑

### 2. 安装依赖

```bash
pip install -r requirements.txt
```

### 3. 启动服务器

在作为服务器的电脑上运行：

```bash
streamlit run main.py --server.port=8501 --server.address=0.0.0.0
```

**参数说明：**
- `--server.port=8501` - 指定端口号（可自定义）
- `--server.address=0.0.0.0` - 允许局域网访问

### 4. 客户端访问

在同一局域网内的其他电脑浏览器中输入：

```
http://[服务器IP地址]:8501
```

例如：`http://192.168.1.100:8501`

**查看服务器IP地址的方法：**

Windows 系统：
```bash
ipconfig
```

找到 "IPv4 地址" 即为服务器IP。

## 📖 功能说明

### 1. 首页
- 系统概览
- 数据统计卡片（员工总数、工地总数、考勤记录数）
- 多语言切换器（顶部）

### 2. 公司管理
- ➕ 新增公司
- 📋 公司列表展示

### 3. 缺勤原因管理
- ➕ 新增缺勤原因
- 📋 缺勤原因列表展示
- ⚠️ 禁止重复添加

### 4. 员工管理
- ➕ 新增员工（姓名 + 所属公司）
- 📋 员工列表展示
- ⚠️ 禁止重复添加同名员工

### 5. 工地管理
- ➕ 添加工地（仅工地名称）
- 📋 工地列表展示
- ⚠️ 禁止重复添加工地

### 6. 考勤记录
- 📝 选择日期、员工
- 🏗️ 上午/下午/晚上工地选择（下拉菜单）
- 🔢 加班小时数输入（0-24小时）
- ⚠️ 同一天同一员工仅允许1条记录
- 💡 规则：只有选择晚上工地时才能记录加班时间
- 📋 考勤记录列表展示

### 7. 员工统计看板
- 🎨 可视化日历视图（行=员工，列=日期）
- 🔵 颜色编码：绿色=出勤，红色=无理由缺勤，黄色=有理由缺勤，灰色=无记录
- 🔍 筛选条件：日期范围（默认最近10天）、公司（多选）、员工（多选）
- 📊 统计摘要：出勤天数、缺勤统计

### 8. 工地统计看板
- 📊 数据透视表（行=工地，列=日期）
- 🔍 筛选条件：日期范围（默认最近10天）、工地（多选）
- 💰 计算规则：上午=0.5天，下午=0.5天，晚上=加班小时数
- 📈 总计行：显示每天所有工地的总工人数
- ✨ 加粗高亮：总计行使用粗体+深色背景

### 9. 个人员工统计
- 👤 单员工考勤记录查询
- 🔍 筛选条件：员工（单选，必填）、日期范围（默认最近10天）
- 📋 详细记录表格：ID、姓名、公司、上午/下午/晚上工地、缺勤原因、日期、加班小时
- 💡 提示：必须选择员工后才显示数据

## 🗄️ 数据库结构

系统使用 SQLite 数据库，自动生成 `attendance.db` 文件。

### companies 表（公司）
```sql
id (INTEGER, 主键)
name (TEXT, 公司名称，唯一)
```

### absent_reasons 表（缺勤原因）
```sql
id (INTEGER, 主键)
reason (TEXT, 缺勤原因，唯一)
```

### workers 表（员工）
```sql
id (INTEGER, 主键)
name (TEXT, 员工姓名，唯一)
company_id (INTEGER, 外键 -> companies.id)
```

### sites 表（工地）
```sql
id (INTEGER, 主键)
name (TEXT, 工地名称，唯一)
```

### attendance 表（考勤）
```sql
id (INTEGER, 主键)
worker_id (INTEGER, 外键 -> workers.id)
morning_site_id (INTEGER, 外键 -> sites.id, 可空)
afternoon_site_id (INTEGER, 外键 -> sites.id, 可空)
evening_site_id (INTEGER, 外键 -> sites.id, 可空)
absent_reason_id (INTEGER, 外键 -> absent_reasons.id, 可空)
work_date (DATE, 工作日期)
overtime_hours (FLOAT, 加班小时)
UNIQUE(worker_id, work_date) - 防止重复提交
```

**重要规则：**
- 上午/下午/晚上工地至少选择一个
- 只有选择晚上工地时才能记录加班时间
- 未选择晚上工地时，加班时间自动设为0

## 🔧 常见问题

### Q1: 如何让其他电脑访问系统？
A: 确保所有电脑在同一局域网内，启动时添加 `--server.address=0.0.0.0` 参数，其他电脑通过 `http://服务器IP:端口` 访问。

### Q2: 数据会同步吗？
A: 是的，所有设备共享同一个 SQLite 数据库文件，数据实时同步。

### Q3: 可以同时在多台电脑上操作吗？
A: 可以，系统支持多用户并发操作。

### Q4: 需要联网吗？
A: 不需要，系统纯本地运行，只需局域网连接。

### Q5: 数据库文件在哪里？
A: 自动在项目根目录生成 `attendance.db` 文件。

### Q6: 如何切换语言？
A: 在页面顶部的语言选择器中选择 English、简体中文 或 繁體中文。

### Q7: 旧数据库如何升级？
A: 系统会自动检测并升级数据库结构，无需手动操作。首次运行时会自动添加缺失的列。

### Q8: 员工统计看板中的颜色代表什么？
A: 绿色=出勤工地，红色=无理由缺勤，黄色=有理由缺勤，灰色=无记录。

### Q9: 工地统计的总工人数如何计算？
A: 上午出勤=0.5天，下午出勤=0.5天，晚上出勤=加班小时数。总计行显示每天所有工地的总和。

### Q10: 为什么个人员工统计页面是空白的？
A: 必须先选择一个员工才会显示数据。这是设计行为，避免加载大量无用数据。
