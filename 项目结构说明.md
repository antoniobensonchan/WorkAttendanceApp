## 🔗 文件关系说明

### 依赖关系
```
main.py (主程序)
  ├── 导入 database.py 的 DatabaseManager
  ├── 加载 style.css 的样式
  └── 使用 Streamlit 渲染界面

database.py (数据库)
  ├── 使用 sqlite3 标准库
  └── 被 main.py 调用

style.css (样式)
  └── 被 main.py 加载应用
```

### 数据流向
```
用户操作 (浏览器)
    ↓
Streamlit 界面 (main.py)
    ↓
数据库操作 (database.py)
    ↓
SQLite 数据库 (attendance.db)
    ↓
返回结果
    ↓
界面更新
```

## 🎯 核心模块说明

### 1. main.py - 主程序
**职责**：界面渲染、用户交互、页面导航

**主要函数**：
- `load_css()` - 加载Apple风格CSS
- `sidebar_menu()` - 侧边栏导航
- `home_page()` - 首页
- `workers_page()` - 员工管理
- `sites_page()` - 工地管理
- `attendance_page()` - 考勤记录
- `worker_stats_page()` - 员工统计
- `site_stats_page()` - 工地统计
- `main()` - 主入口

### 2. database.py - 数据库模块
**职责**：数据持久化、业务逻辑

**核心类**：
- `DatabaseManager` - 数据库管理器（单例模式）
  - `_init_database()` - 初始化表结构
  - `add_worker()` / `get_all_workers()` - 员工管理
  - `add_site()` / `get_all_sites()` - 工地管理
  - `add_attendance()` / `get_all_attendance()` - 考勤管理
  - `get_worker_statistics()` - 员工统计
  - `get_site_statistics()` - 工地统计

### 3. style.css - 样式文件
**职责**：Apple极简UI风格定义

**核心样式**：
- `.button-primary` - 蓝色主按钮（#0071e3）
- `.button-secondary` - 灰色次按钮
- `.card` - 卡片容器
- `table` - 极简表格
- `.sidebar` - 侧边栏
- 响应式媒体查询

## 🔄 数据流程图

### 添加员工流程
```
用户输入姓名和公司
        ↓
点击「保存」按钮
        ↓
检查字段是否为空
        ↓
检查员工是否已存在
        ↓
插入数据库
        ↓
显示成功提示
        ↓
刷新页面
```

### 提交考勤流程
```
选择日期、员工、工地
        ↓
勾选出勤时段
        ↓
输入加班小时
        ↓
点击「提交」
        ↓
检查是否重复提交
        ↓
插入考勤记录
        ↓
显示成功提示
        ↓
自动刷新数据
```

## 🚀 性能特点

### 优势
- ✅ 单文件数据库，便于管理
- ✅ 无网络延迟，响应迅速
- ✅ 局域网访问，速度快
- ✅ 纯本地运行，数据安全

### 限制
- ⚠️ SQLite并发性能有限（适合小型团队）
- ⚠️ 需要同一局域网
- ⚠️ 依赖服务器电脑开机
